### Integration Findings

1. **Admin reminder policy is not wired into student reminder preference/runtime defaults**
   - Affected: POLICY-01, NOTIF-03
   - `get_or_create_student_notification_preference` hardcodes reminder defaults instead of reading `SchoolPrivacyPolicyDefault`.
   - `update_student_notification_preference` applies payload quiet hours directly without policy/default relationship.

2. **Operations audit API exposes raw resource IDs for v1.4 privacy events**
   - Affected: OPS-02, SHARE-05, ACCESS-04
   - `AuditEventItem` includes `resource_id`.
   - `_audit_event_item` returns `event.resource_id` unchanged.
   - Adult summary/shared-note read audits can use student UUIDs as `resource_id`.

3. **Admin note-sharing toggle is not enforced by student share service**
   - Affected: POLICY-01, SHARE-01, SHARE-04
   - Classified as warning/tech debt because explicit v1.4 requirements require admin policy configuration but do not clearly require the note-sharing toggle to disable already-scoped student consent flows.
   - The UI/API exposes `note_sharing_enabled`, so leaving it unenforced is misleading and should be closed with the other policy-runtime gap.
