    - Replace fetch handler with differentiated strategy:
      1. /api/* → return early (privacy guard, unchanged)
      2. fonts.googleapis.com / fonts.gstatic.com → stale-while-revalidate into FONTS_CACHE
      3. /_next/static/* → cache-first with runtime put into CACHE_NAME (hashed = immutable)

      5. All other static → cache-first with silent catch

    CREATE frontend/scripts/inject-sw-assets.mjs:
    - ESM module with node: protocol imports (readFileSync, writeFileSync, readdirSync from 'node:fs', path from 'node:path', fileURLToPath from 'node:url')
    - Resolve frontendRoot via path.dirname(fileURLToPath(import.meta.url)) + path.resolve(scriptDir, '..')
    - Read .next/BUILD_ID to get buildId
    - Read .next/static/{buildId}/_buildManifest.js
    - Extract chunk paths for "/" route via regex: /"\/":\s*\[([^\]]+)\]/
    - Read .next/static/chunks/ directory, filter files starting with 'framework-', 'main-', or matching /^webpack-/
    - Build assetPaths array with: buildManifest path, ssgManifest path, root page chunks (prefixed /_next/), framework/main/webpack chunks (prefixed /_next/static/chunks/)
    - Read public/sw.js, replace SHELL_ASSETS array via regex /const SHELL_ASSETS = \[[\s\S]*?\];/
    - Replace CACHE_NAME value with timestamp-versioned 'peerlight-shell-v' + Date.now()
    - Write updated sw.js back
    - Console.log status: asset count and new cache version name
    - Exit with process.exit(1) if .next/BUILD_ID not found (graceful error message)

    MODIFY frontend/package.json:
    - Change "build" script from "next build" to "next build && node scripts/inject-sw-assets.mjs"
