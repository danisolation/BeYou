<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student browser -> student share API | Student-controlled adult IDs, check-in ID, share scope, and optional summary cross into backend mutation routes. |
| backend share API -> database | Raw private note and student-authored summary may be stored/read; authorization and audit filtering must prevent leakage. |
| adult browser -> adult support-summary API | Adult route requests can enumerate student IDs; backend must enforce active relationship plus active grant for raw shared content. |
| backend service -> audit log | Sensitive operation metadata crosses into audit storage; raw private content, names, emails, contact identifiers, and free-text reasons must not enter audit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Spoofing | `POST /api/student/mood-check-ins/{checkin_id}/shares` | mitigate | Require cookie session via `get_current_user`, `require_role(student)`, privacy acknowledgement, and same-site mutation guard before mutation. |
| T-23-02 | Tampering | `MoodNoteShareRequest.adult_ids/share_scope/student_summary` | mitigate | Validate Pydantic schema, normalize summary, reject empty adult_ids, reject invalid scope, load check-in by current student, and validate adults against active `StudentAdultLink`. |
| T-23-03 | Repudiation | share/revoke/read operations | mitigate | Record metadata-only audit for `mood_note_share_created`, `mood_note_share_updated`, `mood_note_share_revoked`, and `mood_note_share_read` with IDs/status/scope only. |
| T-23-04 | Information Disclosure | adult support summary shared notes | mitigate | Require active relationship plus active `MoodNoteShare.revoked_at IS NULL`; relationship-only `shared_mood_note` permission is insufficient; revoked shares are filtered immediately. |
| T-23-05 | Information Disclosure | audit metadata | mitigate | Use `record_audit_event`; metadata keys must exclude `private_note`, `student_summary`, `shared_note_text`, raw content, free-text reason, names, emails, and contact identifiers. |
| T-23-06 | Denial of Service | repeated share submissions | mitigate | Re-share existing active `(mood_checkin_id, adult_id)` updates existing grant instead of creating duplicate active grants; rely on existing active unique index. |
| T-23-07 | Elevation of Privilege | adult role path mismatch | mitigate | Teacher route requires teacher role and teacher relationship; parent route requires parent role and parent relationship; adult shared-note service checks relationship_type route value and active grant. |
| T-23-08 | Information Disclosure | frontend adult selection and shared-note display | mitigate | Student UI displays adult name/relationship for confirmation only; no email/contact identifiers; adult UI separates shared content from aggregate trend and never expands raw mood history. |
| T-23-09 | Tampering | CSRF on share/revoke | mitigate | All POST/DELETE share routes call `require_same_site_mutation(request, settings)`. |
| T-23-10 | Privacy Side Effect | notifications/SOS/risk/background delivery | mitigate | Do not touch `SosAlert`, `InAppNotification`, external messages, risk scores, workers, or background tasks; tests assert counts unchanged and grep gates enforce. |
</threat_model>
