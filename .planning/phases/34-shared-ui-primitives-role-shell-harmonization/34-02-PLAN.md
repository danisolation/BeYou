<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| browser → authenticated layout | Untrusted route path determines expected role surface. |
| auth API → layout | Current user role/privacy flags drive child rendering. |
| role shell → children | Shell must block children for wrong-role/privacy states. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-34-05 | Elevation of Privilege | `AuthenticatedLayout` | mitigate | Keep expected-role and privacy gate logic in layout; test children blocked before render. |
| T-34-06 | Information Disclosure | wrong-role/privacy surfaces | mitigate | Do not render route children until role/privacy checks pass. |
| T-34-07 | Spoofing/Session | logout/auth flow | mitigate | Preserve `getCurrentUser` and cookie-authenticated `apiFetch` logout; static no-token-storage checks. |
| T-34-08 | Denial of Accessibility | shell navigation | mitigate | Preserve skip link, `aria-current`, overflow nav, and `min-h-11` tests. |
</threat_model>
