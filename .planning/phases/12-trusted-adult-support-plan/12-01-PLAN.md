# 12-01 Plan: Support Plan Backend Domain, APIs, Authorization, and Audit

<objective>
Add student-owned support plan persistence and APIs with linked-adult validation and metadata-only audit.
</objective>

## Tasks

1. Add support plan lifecycle enum, models, relationships, indexes, and Alembic migration.
2. Add schemas for support-plan payloads, linked adults, selected adults, and response state.
3. Add service logic for loading/upserting support plans, validating active linked adults, preserving historical records, and recording safe audit metadata.
4. Add student-only API routes under `/api/student/support-plan`.
5. Add backend tests for create/update/deactivate, linked-adult validation, role access, and audit metadata privacy.
