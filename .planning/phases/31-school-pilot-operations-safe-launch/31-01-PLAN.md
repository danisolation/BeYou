<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| readiness/config/database -> admin operations API | System and DB state become operator-visible launch metadata. |
| admin operations API -> authenticated admin browser | Launch metadata is rendered to operators without raw records. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-31-01-01 | I | pilot launch evidence | mitigate | Evidence strings use only statuses, booleans, counts, and safe command names; tests reject raw emails, IDs, notes, transcripts, answers, claims, tokens, and provider subjects. |
| T-31-01-02 | E | launch checklist interpretation | mitigate | Checklist is read-only metadata and does not create approvals, roles, launch toggles, or provider claim authorization. |
| T-31-01-03 | S | demo state in pilot | mitigate | Runtime and demo seed/login checks fail when production pilot is unsafe. |
| T-31-01-04 | R | operator proof | mitigate | Privacy regression is represented as a command-backed metadata item; no raw test logs are stored in the response. |
</threat_model>
