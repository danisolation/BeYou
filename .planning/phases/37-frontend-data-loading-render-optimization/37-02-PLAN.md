<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Student dashboard -> profile API | Profile is primary trust data and must fail loudly. |
| Student dashboard -> SOS/reminder APIs | Supplemental reads may fail locally without becoming false success. |
| API-provided href -> browser navigation | Reminder navigation must pass through `safeInternalHref`. |
| Student private data -> browser storage | No tokens, profile, reminders, SOS payloads, or API responses are persisted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-37-02-01 | Information Disclosure | Student profile | high | mitigate | Profile read remains primary; dashboard shows `ErrorState` on failure and does not guess profile values in skeleton. |
| T-37-02-02 | Safety UX | SOS history | medium | mitigate | SOS read failure shows `STUDENT_SOS_UNAVAILABLE_MESSAGE` instead of silently claiming no SOS history. |
| T-37-02-03 | Safety UX | mood reminder | medium | mitigate | Reminder read failure states that no adult notification or SOS is created by the load failure. |
| T-37-02-04 | Spoofing/Tampering | reminder href | high | mitigate | Existing `safeInternalHref(moodReminder.href)` flow remains tested. |
| T-37-02-05 | Information Disclosure | browser storage | high | mitigate | Tests reject browser storage and token markers in Student dashboard sources. |
</threat_model>
