# Phase 37 - UI Design Contract

> Visual and interaction contract for frontend data loading and render optimization. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - preserve existing manual Tailwind Peerlight AI primitives |
| Preset | not applicable |
| Component library | none |
| Icon library | lucide-react |
| Font | Inter, ui-sans-serif, system-ui |

Phase 37 must reuse the Phase 34 presentation primitives in `frontend/components/ui-primitives.tsx`. Do not initialize shadcn, do not add a new UI framework, and do not introduce a global React Query provider by default.

---

## Spacing Scale

Declared values must remain multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline gaps |
| sm | 8px | Compact nav/card internal gaps |
| md | 16px | Default element spacing and form padding |
| lg | 24px | Section/card desktop padding |
| xl | 32px | Page grid gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Existing `p-5` card surfaces may remain when touched only for loading/render behavior; new skeleton/placeholder surfaces should prefer `p-4` mobile and `p-6` desktop.
- All actionable links/buttons must preserve `min-h-11` / 44px minimum touch target.
- Scoped skeleton rows may use 8px inner vertical rhythm only inside already bounded cards.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 or 600 | 1.5 |
| Heading | 20px | 600 | 1.25 |
| Display | 36px | 600 | 1.12 |

Only weights 400 and 600 are allowed in new or touched Phase 37 UI copy and loading states. Existing Phase 34 display primitives may retain their legacy weight token when reused unchanged; Phase 37 must not add new font-weight values for loading states.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FCFF` background + `#FFFFFF` surfaces | App background, dashboard cards, loading containers |
| Secondary (30%) | `#EAF7F3` | Privacy boundary cards, scoped skeleton panels, grouped supportive content |
| Accent (10%) | `#2CA58D` | Primary safe CTAs, active nav, Peerlight AI label, focus treatment, safe status copy |
| Warning | `#F59E0B` | Optional caution/unavailable panel state only |
| Destructive/SOS | `#DC2626` / red-600/red-700 | SOS, high-risk, primary load failures, true error alerts only |

Accent reserved for: primary safe CTA, active role/nav state, Peerlight brand label, safe status badge, focus outline, supportive non-danger links.

Red/destructive reserved for: Student SOS actions, urgent/high-risk status, `ErrorState`, `role="alert"` primary failures, SOS confirmation/error copy. Loading skeletons must never use red unless representing a known SOS/high-risk semantic already returned by data.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Preserve role-specific CTAs: Student `Gửi SOS hỗ trợ`; Teacher `Xem và cập nhật SOS`; Parent `Xem trạng thái SOS`; Admin `Mở bảng vận hành metadata` |
| Loading state | `Đang tải thông tin...` for generic state; scoped panels may use `Đang tải tóm tắt hỗ trợ...`, `Đang tải thông báo hỗ trợ...`, or `Đang tải metadata vận hành...` |
| Empty state heading | Must distinguish true empty from unavailable. Student `Chưa có tín hiệu SOS nào`; Teacher/Parent `Chưa có học sinh SOS được phép xem`; Admin `Chưa có metadata vận hành để hiển thị` |
| Empty state body | Student `Khi cần, em có thể gửi tín hiệu để người lớn tin tưởng biết em cần hỗ trợ.` Teacher `Khi có liên kết và tín hiệu SOS được phép xem, tóm tắt hỗ trợ sẽ hiển thị tại đây.` Parent `Peerlight AI chỉ hiển thị tóm tắt được phép xem, không mở nội dung riêng tư thô của con.` Admin `Bảng này chỉ hiển thị metadata tổng hợp hoặc preview bounded, không mở dữ liệu riêng tư thô.` |
| Primary error state | `Chưa tải được thông tin. Hãy thử lại hoặc quay về cổng phù hợp để tiếp tục an toàn.` |
| Optional panel unavailable state | `Mục này tạm thời chưa tải được. Các phần còn lại vẫn giữ nguyên phạm vi quyền riêng tư.` |
| Admin bounded-count wording | Use honest preview wording, e.g. `Preview tài khoản demo` / `Preview liên kết được phân trang`; do not imply exact totals when only bounded lists are loaded |
| Destructive confirmation | No new destructive reset/export controls. Preserve existing SOS confirmation: `Xác nhận gửi tín hiệu hỗ trợ` / `Ở lại trang này` |

Copy must remain Vietnamese-first, supportive, concise, non-diagnostic, and role-specific.

## Phase 37 Loading and Interaction Contract

### Dashboard-level loading

- Static dashboard structure may render before supplemental data only when it does not guess sensitive values:
  - `PageHeader`
  - `PrivacyBoundaryCard`
  - demo guide rhythm
  - safe role boundary guidance
- Sensitive API values must not be guessed, prefilled, or shown before loading completes.
- Primary dashboard data that gates trust must fail to `ErrorState` with `role="alert"`.

### Scoped skeletons/placeholders

Use Phase 34 primitives only:
- `LoadingState` for generic route-level loading.
- `SurfaceCard` + muted placeholder rows for scoped section loading.
- `EntryCard` rhythm for card-grid skeletons.
- `StatusBadge` only when the status is known; do not show fake status badges.

Required skeleton rhythm:
- Card radius: `rounded-3xl`.
- Surface: white or secondary.
- Padding: 16px mobile, 24px desktop.
- Gap: 16px between skeleton rows/cards.
- Accessible announcement: `role="status"` and `aria-live="polite"` on the scoped loading container.

### Optional panel failures

Optional data must degrade locally, not collapse the whole page:
- Student SOS history/reminder failures: show scoped unavailable copy or omit only with explicit safe copy.
- Teacher/Parent support overview failure: keep linked-student visibility honest; do not filter students away as if no SOS-visible students exist.
- Teacher/Parent notification failure: show `Thông báo hỗ trợ tạm thời chưa tải được.` instead of a successful "no notifications" state.
- Admin count/preview failure: show `Preview metadata tạm thời chưa tải được.` and keep metadata-only entry cards available.

### Adult dashboard distinction

Teacher/Parent presentation must distinguish:
1. no linked/SOS-visible students,
2. support overview unavailable,
3. notification panel unavailable,
4. true primary dashboard load failure.

Do not use a single empty state for all four cases.

### Student dashboard distinction

Student dashboard must preserve:
- profile as primary trust gate,
- SOS red/destructive visual priority,
- mood reminder as optional scoped panel,
- SOS history as scoped list,
- privacy card before sensitive dashboard interpretation.

### Admin dashboard distinction

Admin dashboard must preserve metadata-only posture:
- Do not fetch broad `/api/admin/users` or `/api/admin/links` only to count totals.
- If exact totals are unavailable, use bounded/preview wording.
- Do not add raw exports, raw audit browsers, risk leaderboards, per-student drilldowns, or destructive reset controls.

## Privacy, Cache, and Auth Boundary Contract

- `frontend/app/(authenticated)/layout.tsx` remains the single owner of:
  - `getCurrentUser`
  - privacy acknowledgement redirects
  - wrong-role handling
  - logout
  - role shell
  - no-token auth behavior
- Dashboard API reads must remain cookie-authenticated with `credentials: "include"`.
- Credentialed dashboard reads must be explicit `no-store` by default.
- Do not store tokens, role data, support summaries, notifications, raw payloads, or API responses in:
  - `localStorage`
  - `sessionStorage`
  - IndexedDB
  - service workers
  - unscoped global caches
- If in-memory request dedupe is introduced, keys must include effective user/role boundary and request path/query.
- Reason-gated, relationship-gated, policy-sensitive, runtime-sensitive, or logout-sensitive data must default to no-store unless a narrowly scoped invalidation rule is proven.
- Preserve `safeInternalHref` for API-provided navigation targets.

---

## Bundle and Import Contract

- Shared presentation primitives must remain presentation-only and must not import:
  - route pages
  - API clients
  - auth helpers
  - role services
  - business-rule logic
- Shared dashboard loading helpers may live under neutral `frontend/lib/` paths.
- Parent must not import Teacher route pages.
- No role page may import another role page.
- Next build output must be reviewed for affected `/student`, `/teacher`, `/parent`, `/admin`, and touched subroutes.
- Evidence must remain aggregate-only: route labels, source files, request counts, build asset counts/bytes, command names, pass/fail status.

## Role-Specific Contract

### Student

- Preserve Student-first privacy/support tone.
- Preserve red SOS visual priority and confirmation copy.
- Loading optimization must not weaken SOS confirmation, success, failure, or linked-adult privacy copy.
- Mood reminder unavailable state must not imply adult notification or SOS creation.

### Teacher

- Preserve teacher-specific SOS coordination posture.
- Support overview and notifications may fail independently.
- Teacher can see and update SOS status only through existing permitted routes.
- Do not expose raw self-check answers, private notes, chat transcripts, raw identifiers beyond existing allowed display, or per-student risk drilldowns.

### Parent

- Preserve read-only supportive posture.
- Parent can view permitted SOS/support summaries only.
- Parent unavailable states must not imply action authority.
- Do not add teacher-style update controls.

### Admin

- Preserve metadata-only operations posture.
- Count cards must use bounded/preview wording unless backed by safe metadata totals.
- Do not add raw exports, destructive reset controls, risk leaderboards, raw audit browsers, or per-student drilldowns.
