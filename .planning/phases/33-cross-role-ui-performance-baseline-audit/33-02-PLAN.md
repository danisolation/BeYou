Create repeatable test-side frontend/backend performance baseline helpers and the Phase 33 aggregate-only performance baseline artifact.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Test client to backend API | Backend endpoints are exercised through test-side helpers and may return sensitive role-scoped data. |
| Database/session fixtures to aggregate metrics | Test data and SQL activity are converted into timing, payload byte, and query-count candidate metadata. |
| Frontend route files/build output to baseline artifact | Static route/build evidence is summarized without copying runtime user data. |
| Helper stdout to planning artifact | Console output can influence the baseline artifact and must remain aggregate-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-33-02-01 | Information Disclosure | Backend baseline helper | mitigate | Compute payload byte size in memory and record only byte counts/status categories/durations/query-count candidates; never print or persist raw bodies. |
| T-33-02-02 | Elevation of Privilege | Authenticated endpoint measurement | mitigate | Use existing test fixtures and preserve backend role checks, active relationships, SOS-only adult visibility, reason gates, audit metadata, and admin metadata sanitizers. |
| T-33-02-03 | Tampering | Baseline evidence | mitigate | Record command/source for every row and avoid arbitrary thresholds; findings remain candidates until optimization phases. |
| T-33-02-04 | Denial of Service | Measurement commands | mitigate | Use bounded local test-side measurements only; do not run load tests or stress production/demo endpoints. |
| T-33-02-05 | Information Disclosure | Frontend build/helper output | mitigate | Output only route names, source files, aggregate fetch counts, build evidence availability, and command names; no browser tokens, cookies, storage, headers, or user values. |
| T-33-02-06 | Spoofing | Environment labels | mitigate | Distinguish local deterministic, public demo, Render/Vercel cold/warm, and live production-pilot unavailable constraints; never label public demo as pilot proof. |
</threat_model>
