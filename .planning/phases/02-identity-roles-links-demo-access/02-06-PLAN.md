<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Admin UI → admin backend APIs | Privileged account/link mutations initiated by browser. |
| Admin UI forms → backend validation | Client validation supports but does not replace backend authorization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-05-01 | S | admin forms | mitigate | UI requires authenticated admin layout; backend still enforces admin role. |
| T-02-05-02 | I | password create field | mitigate | Password used only for create request and not rendered after submit or stored locally. |
| T-02-05-03 | T | admin mutations | mitigate | Use backend cookie-auth endpoints protected by CSRF Origin/Fetch Metadata checks. |
| T-02-05-04 | E | link form | mitigate | UI restricts role selectors; backend validates exact roles and relationship type. |
| T-02-05-05 | I | admin error display | mitigate | Generic error copy only; no backend stack traces or secrets displayed. |
| T-02-05-06 | R/I | demo delete/link rows | mitigate | `Demo` badge and exact demo warning copy required for demo-only delete. |
</threat_model>
