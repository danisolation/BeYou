## Threat Model

| Threat | Mitigation |
|---|---|
| Admin enables external reminders before governance exists | Schema and service reject external channels and `external_channels_enabled=True`. |
| Unknown/raw-note policy field is silently accepted | Policy update schema forbids extra fields. |
| Operations dashboard leaks raw content | Existing sanitizer remains; tests assert raw notes, reason text, emails, identifiers, exports, and drilldowns are absent. |
| Policy controls become surveillance tooling | UI copy and data model expose defaults/counts only, no per-student risk views. |
| Demo readiness exposes student content | Demo seed readiness returns counts/statuses only. |

## Success Criteria

- POLICY-01, POLICY-02, OPS-01, OPS-02, and OPS-03 are satisfied.
- Existing Phase 21-24 privacy behavior remains intact.
- No external notification delivery, raw exports, risk score dashboards, or free-text reason narratives are introduced.
