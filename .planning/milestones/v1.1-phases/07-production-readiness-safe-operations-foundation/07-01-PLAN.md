# 07-01 Plan: Readiness Service and API Contracts

<objective>
Add backend readiness schemas, service checks, and public/admin endpoints that distinguish liveness from production readiness without exposing secrets.
</objective>

## Tasks

1. Create readiness response schemas with stable overall/check statuses.
2. Implement a readiness service covering environment/config hygiene, database connectivity, Alembic head/current state, cookie/origin safety, and provider-secret readiness.
3. Add `/health/live`, `/health/ready`, and `/api/admin/operations/readiness`.
4. Include the new admin operations router from `create_app`.

## Acceptance

- `/health` remains backwards compatible.
- `/health/live` returns cheap liveness.
- `/health/ready` returns only overall non-sensitive readiness.
- Admin readiness returns detailed checks with remediation hints and no raw secrets.
