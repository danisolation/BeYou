<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| admin session -> admin list APIs | Only active Admin users with `admin_operations` permission may list accounts/links. |
| admin list API -> response body | Existing safe response fields remain; password/secret fields never serialize. |
| joined DB query -> link response | Student/adult hydration happens in one joined query without broad raw-data exposure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-36-02-01 | Information Disclosure | admin user/link lists | mitigate | Server-side bounds default 100/max 200 and response redline tests reject password/token/private-note markers. |
| T-36-02-02 | Elevation of Privilege | admin routes | mitigate | Keep `_require_admin` unchanged and add student-client 403 regression assertions. |
| T-36-02-03 | Denial of Service | unbounded list queries | mitigate | FastAPI `Query` constraints reject `limit=201` and services apply SQL `limit/offset`. |
| T-36-02-04 | Tampering | link hydration | mitigate | Joined hydration preserves existing fields and mutation paths still use validated create/revoke services. |
</threat_model>
