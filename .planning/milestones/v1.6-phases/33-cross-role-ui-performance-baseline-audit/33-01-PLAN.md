<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Source code to audit artifact | Production UI files are inspected and summarized into a planning artifact. |
| Sensitive UI fields to evidence notes | Legitimate UI fields may represent sensitive student/adult/admin data and must be classified, not copied. |
| Human/manual observation to artifact | Optional visual/manual notes enter the artifact and must remain aggregate-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-33-01-01 | Information Disclosure | `33-UI-INVENTORY.md` | mitigate | Record only aggregate route/state/pattern/severity/file-reference metadata. Classify sensitive fields instead of copying values per D-09 through D-11. |
| T-33-01-02 | Tampering | UI inventory findings | mitigate | Require file references and repeatable route/state coverage helper so findings are traceable to inspected source files. |
| T-33-01-03 | Repudiation | Manual audit notes | mitigate | Include environment/evidence labels and file references for every finding; avoid unverifiable narrative-only claims. |
| T-33-01-04 | Elevation of Privilege | Role boundary interpretation | mitigate | Preserve existing role checks, privacy acknowledgement behavior, SOS-only adult visibility, reason gates, and metadata-only admin boundaries; do not change production auth or UI logic. |
| T-33-01-05 | Denial of Service | Inventory test helper | accept | Helper performs static/local checks only and does not call production services or stress endpoints. |
</threat_model>
