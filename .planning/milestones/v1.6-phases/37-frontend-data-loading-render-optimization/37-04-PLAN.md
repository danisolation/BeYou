## Trust Boundaries

| Boundary | Description |
|---|---|
| Admin dashboard -> Admin user/link APIs | Preview reads must be bounded/no-store and not represented as exact totals. |
| preview failure -> Admin UI | Preview failure must stay local and not erase metadata-only operations navigation. |
| Admin metadata UI -> private student data | Admin dashboard must not add raw data exports/drilldowns/audit browsers/reset controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-37-04-01 | Information Disclosure | Admin preview reads | high | mitigate | Use `dashboardRead` with bounded `limit=10` URLs and no-store options. |
| T-37-04-02 | Tampering | Admin count labels | medium | mitigate | Labels say `Preview ...` and `liên kết được phân trang`, never exact totals. |
| T-37-04-03 | Denial of Service | Admin dashboard | medium | mitigate | Preview failures render scoped unavailable copy while keeping metadata entry cards available. |
| T-37-04-04 | Information Disclosure | Admin controls | high | mitigate | Tests reject exports, risk leaderboards, drilldowns, raw audit, and reset labels. |
| T-37-04-05 | Elevation of Privilege | Admin API mutations | high | mitigate | Mutation helpers remain unchanged and no new destructive controls are added. |
</threat_model>
