<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| adult dashboard -> linked-students API | Linked-student lists are primary, role-owned data gates. |
| adult dashboard -> support overview API | Support overview is optional for rendering linked students but sensitive when available. |
| adult dashboard -> notifications API | Notifications are optional scoped panels and unsafe hrefs must be sanitized. |
| Teacher route -> Parent route | Role pages must not import each other. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-37-03-01 | Information Disclosure | adult support overview | high | mitigate | Support overview uses no-store helper and does not persist or cache response data. |
| T-37-03-02 | Safety UX | adult empty states | high | mitigate | Component distinguishes no linked students, no SOS-visible students, support unavailable, and primary failure. |
| T-37-03-03 | Spoofing/Tampering | notification hrefs | high | mitigate | `safeInternalHref(notification.href)` remains the only rendering path for API hrefs. |
| T-37-03-04 | Elevation of Privilege | Parent dashboard | high | mitigate | Parent page keeps read-only copy and does not import Teacher page or Teacher update controls. |
| T-37-03-05 | Information Disclosure | cross-role code sharing | high | mitigate | Shared logic lives in neutral `frontend/lib/` and tests reject role-page-to-role-page imports. |
</threat_model>
