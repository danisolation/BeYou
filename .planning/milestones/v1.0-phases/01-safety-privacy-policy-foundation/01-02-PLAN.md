<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| sensitive_event_to_audit_log | Runtime events must be reduced to metadata before audit storage. |
| demo_data_to_real_data | Demo records must be visibly and structurally distinguished from real student data. |
| safety_signal_to_adult_summary | Safety signals must not expose unnecessary raw content to adults. |

## Assets

`student_self_check_raw_answers`, `self_check_summary`, `chat_transcript_raw`, `chat_safety_signal`, `sos_alert`, `student_adult_link`, `audit_event`, `demo_record`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| raw_sensitive_content_in_audit_log | Information Disclosure | 01-AUDIT-EVENT-CATALOG.yml | mitigate | Define `audit_policy: metadata_only` and `forbidden_fields` including raw chat, raw self-check answers, passwords, tokens, and secrets. |
| demo_record_mistaken_for_real_student | Tampering / Information Disclosure | 01-DEMO-DATA-POLICY.md | mitigate | Require `is_demo`, persistent banner, and `Demo` badge; never rely only on color. |
| idor_access_to_unlinked_student | Information Disclosure / Elevation of Privilege | audit records | mitigate | Audit sensitive reads and link changes with actor/resource metadata. |
</threat_model>
