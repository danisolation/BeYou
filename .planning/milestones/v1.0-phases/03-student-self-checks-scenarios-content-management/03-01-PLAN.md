<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| browser session -> backend authorization | Cookie-authenticated users request sensitive student resources. |
| backend -> PostgreSQL | Sensitive wellbeing records and raw answers become durable data. |
| admin content -> student attempts | Mutable content is copied into immutable attempt snapshots. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-01-01 | I | `self_check_attempt_answers` | mitigate | Store raw answers only in student-scoped answer table; adult/admin schemas omit raw answer fields. |
| T-03-01-02 | E | `require_permission` | mitigate | Add explicit resource checks and keep final deny-by-default raise for unmatched Phase 3 resources. |
| T-03-01-03 | T | attempt snapshots | mitigate | Add snapshot columns to attempt tables so future content edits cannot alter historical records. |
| T-03-01-04 | I | audit metadata | mitigate | Reuse existing forbidden audit metadata keys; no task adds raw answers to audit schemas. |
| T-03-01-05 | T | demo/real data | mitigate | Add `is_demo` on all Phase 3 content and attempt tables, preserving demo policy. |
</threat_model>
