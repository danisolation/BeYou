## Trust Boundaries
| Boundary | Description |
|---|---|
| student browser -> scenario API | Student-submitted choice IDs are untrusted. |
| scenario content -> attempt snapshots | Mutable content is copied into historical records. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-03-01 | T | `submit_scenario_attempt` | mitigate | Verify selected choice belongs to the requested published scenario. |
| T-03-03-02 | I | scenario history | mitigate | Query history by current student only and do not add adult scenario endpoints in Phase 3. |
| T-03-03-03 | E | scenario routes | mitigate | Require student role plus privacy acknowledgement before access. |
| T-03-03-04 | T | content edits | mitigate | Persist feedback/recommendation/lesson/skill snapshots at attempt time. |
| T-03-03-05 | R | attempts | mitigate | Store completed timestamp and `is_demo` on every scenario attempt. |
</threat_model>
