<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| admin browser -> admin content API | Privileged content mutations cross into backend. |
| demo seed -> database | Demo records are inserted into real tables. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-05-01 | CSRF | admin content mutations | mitigate | Call `require_same_site_mutation` for every POST/PATCH/DELETE. |
| T-03-05-02 | T | publish validation | mitigate | Block invalid published content with backend validation, not UI-only checks. |
| T-03-05-03 | R | admin changes | mitigate | Record `admin_safety_content_changed` metadata-only audit event for each mutation. |
| T-03-05-04 | T | historical attempts | mitigate | Archive instead of deleting used content; delete only unused drafts. |
| T-03-05-05 | I | demo/real mixing | mitigate | Seed only when `ALLOW_DEMO_SEED=true` and set `is_demo=True` on content/attempts. |
</threat_model>
