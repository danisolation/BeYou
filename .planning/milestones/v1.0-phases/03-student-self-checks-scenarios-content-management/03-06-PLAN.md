<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| student UI -> backend APIs | Browser collects answer/choice IDs and sends them to backend. |
| backend response -> student UI | Sensitive result/detail data is rendered in browser. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-06-01 | I | self-check detail page | mitigate | Render raw answers only under `/student/self-checks/history/{attemptId}` and no adult/admin links. |
| T-03-06-02 | I | token storage | mitigate | Use `apiFetch` cookie credentials only; no localStorage/sessionStorage. |
| T-03-06-03 | T | form submits | mitigate | Disable pending submit and send only selected UUIDs; backend validates ownership. |
| T-03-06-04 | I | copy language | mitigate | Grep for forbidden medical/surveillance words and keep non-diagnostic copy. |
| T-03-06-05 | S | route role mismatch | transfer | Existing authenticated layout enforces role route UX; backend enforces real authorization. |
</threat_model>
