<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Backend role response → frontend route rendering | UI consumes backend route truth but is not the authorization source. |
| Adult dashboard API response → student profile cards | UI displays only data the backend already relationship-authorized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-05-01 | S | authenticated layout | mitigate | `/api/auth/me` is the session source; unauthenticated users return to login. |
| T-02-05-02 | I | browser storage | mitigate | No localStorage/sessionStorage token writes; `apiFetch` uses cookie credentials. |
| T-02-05-03 | T | logout mutation | mitigate | Use backend cookie-auth logout endpoint protected by SameSite and backend mutation checks. |
| T-02-05-04 | E | wrong role portal | mitigate | Show unauthorized message and correct-dashboard link; backend remains authorization source. |
| T-02-05-05 | I | adult dashboards | mitigate | Render linked-student summaries only; no raw self-check answers, chat transcripts, SOS details, or risk leaderboard. |
| T-02-05-06 | R/I | demo UI | mitigate | Persistent demo banner and `Demo` badge; separation not color-only. |
</threat_model>
