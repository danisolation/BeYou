<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser → auth API | Untrusted credentials and cookie-auth requests enter backend. |
| Auth API → session DB | Backend creates and revokes opaque sessions. |
| Adult portal API → student profile data | Teacher/parent access crosses relationship authorization boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-02-01 | S | POST /api/auth/login | mitigate | Use Argon2id verification, generic failure messages, disabled/deleted account blocking, and rate-limit repeated invalid attempts by normalized email/client IP; tests cover invalid login and HTTP 429 throttle. |
| T-02-02-02 | I | session cookie | mitigate | Dev/E2E uses non-prefixed `beyou_session` over localhost HTTP; production may use `__Host-beyou_session` only with Secure, HttpOnly, SameSite=Lax, Path=/, no Domain; no localStorage token. |
| T-02-02-03 | T | POST /api/auth/logout and POST /api/privacy/acknowledgements | mitigate | Apply Origin and Fetch Metadata checks to mutating cookie-auth endpoints. |
| T-02-02-04 | E | teacher/parent student endpoints | mitigate | Central deny-by-default `require_permission` checks active `student_adult_link`. |
| T-02-02-05 | I | audit/privacy services | mitigate | Metadata-only audit recorder rejects forbidden secret/sensitive keys. |
| T-02-02-06 | R/I | demo sessions/profile responses | mitigate | Propagate `is_demo` in session and profile responses for UI banner/badge. |
| T-02-02-07 | I/T | credentialed browser fetch | mitigate | Exact-origin CORS from `FRONTEND_ORIGIN` with `allow_credentials=True`; no wildcard origin with cookies. |
</threat_model>
