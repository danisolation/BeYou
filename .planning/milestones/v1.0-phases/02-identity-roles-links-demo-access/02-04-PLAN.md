<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser UI → backend APIs | Frontend sends credentials and cookie-auth requests. |
| Backend role response → frontend routing | Login/privacy flow consumes backend route truth but is not authorization source. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-04-01 | S | login form | mitigate | Generic error copy; backend verifies credentials; UI does not reveal account existence. |
| T-02-04-02 | I | browser storage | mitigate | API client uses `credentials: include`; tests forbid localStorage/sessionStorage token writes. |
| T-02-04-03 | T | privacy acknowledgement POST | mitigate | Use backend cookie-auth endpoint; backend CSRF checks protect mutation. |
| T-02-04-04 | E | frontend login redirect | mitigate | UI uses backend login response and `/api/auth/me` route metadata; backend remains source of truth. |
| T-02-04-05 | I | UI diagnostics/errors | mitigate | Show generic Vietnamese errors; do not render stack traces, secrets, raw chat, or raw self-check answers. |
| T-02-04-06 | R/I | demo UI | mitigate | Persistent demo banner and `Demo` badge; separation not color-only. |
</threat_model>
