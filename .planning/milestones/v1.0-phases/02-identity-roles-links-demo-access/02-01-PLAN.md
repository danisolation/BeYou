## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-01-01 | S | users/password_hash schema | mitigate | Store only `password_hash`; use Argon2id in later security service; never create plaintext password column. |
| T-02-01-02 | I | sessions table | mitigate | Store opaque session id server-side; do not create token/localStorage fields. |
| T-02-01-03 | T | mutating API schema foundation | mitigate | Include config for `FRONTEND_ORIGIN`, localhost-safe dev cookie, production `__Host-` cookie guard, and SameSite cookie setup consumed by CSRF checks. |
| T-02-01-04 | E | student_adult_links table | mitigate | Enforce relationship type enum `teacher|parent`, status enum, student/adult FKs for relationship checks. |
| T-02-01-05 | I | audit_events table | mitigate | Audit model has metadata fields only and excludes passwords, tokens, session cookies, API keys, raw chat, raw self-check answers. |
| T-02-01-06 | R/I | demo data markers | mitigate | Add `is_demo` to users, sessions, acknowledgements, links, and audit events. |
</threat_model>
