## Trust Boundaries

| Boundary | Description |
|---|---|
| Admin browser → admin APIs | Privileged mutations enter backend. |
| Seed command → database | Demo account fixture writes identity/link records. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-03-01 | S | admin user create passwords | mitigate | Hash with Argon2id and never echo or audit plaintext password. |
| T-02-03-02 | I | admin session | mitigate | Admin APIs require existing opaque session and admin role, no localStorage token. |
| T-02-03-03 | T | admin mutating endpoints | mitigate | Apply admin role check, purpose `admin_operations`, and explicit `require_same_site_mutation(request)` CSRF Origin/Fetch Metadata checks on POST/PATCH/DELETE. |
| T-02-03-04 | E | admin links | mitigate | Validate student/adult roles and relationship type before creating links. |
| T-02-03-05 | I | audit events | mitigate | Record metadata-only `role_changed`, `account_status_changed`, `student_adult_link_created`, `student_adult_link_revoked`; forbidden keys rejected. |
| T-02-03-06 | R/I | demo seed | mitigate | Gate with `ALLOW_DEMO_SEED=true`; set `is_demo=true` on every seeded user/link. |
</threat_model>
