# Plan 06-01: Backend Aggregate Reports API

## Objective

Create an admin-only aggregate report API with privacy suppression and metadata-only audit.

## Tasks

1. RED: add pytest coverage for admin aggregate report authorization, aggregate domains, suppression, demo filtering, audit, and no raw sensitive fields.
2. GREEN: add schemas, service, router, authorization allow-list entry, and main router inclusion.
3. Verify targeted backend tests and update summary.

## Acceptance criteria

- `GET /api/admin/reports/aggregate` is admin-only.
- Supports `demo_scope=all|demo|real`.
- Includes aggregate counts for users, links, self-check usage/risk, SOS, popular scenarios, and chatbot safety signals.
- Sensitive non-zero buckets below 3 return `count: null` and `suppressed: true`.
- Response and audit metadata omit raw answers, chat transcripts/messages, SOS notes, emails, names, IDs, and freeform sensitive text.
- Successful reads create `sensitive_resource_read` audit event for `aggregate_report`.

## Dependencies

- Existing Phase 1 audit/authorization.
- Existing Phase 3 self-check/scenario tables.
- Existing Phase 4 SOS tables.
- Existing Phase 5 chat safety tables.
