# Phase 05 Plan 03: Review, E2E Smoke, Verification, and Phase Closure

## Objective

Review Phase 05 changes, fix real issues, run automated verification, document UAT status, and update planning state only if Phase 05 passes.

## Requirements Covered

- CHAT-01, CHAT-02, CHAT-03, CHAT-04, CHAT-05, CHAT-06, ADMIN-04

## Tasks

1. **Code review**
   - Review changed backend/frontend files for high-signal bugs/security issues.
   - Create `05-REVIEW.md`.
   - If real issues are found, fix them and create `05-REVIEW-FIX.md`.

2. **Automated verification**
   - Run backend Alembic upgrade and full pytest.
   - Run frontend full Vitest.
   - Run relevant Playwright/E2E if infrastructure is available; otherwise document why not required/blocking.

3. **UAT documentation**
   - Create `05-HUMAN-UAT.md` noting any subjective UI checks not fabricated as approved.
   - If automated checks pass and only optional human polish remains, mark no human blocker.

4. **Phase closure**
   - Create `05-VERIFICATION.md`.
   - Update `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, `.planning/STATE.md`, and `.planning/PROJECT.md` for Phase 05 completion only if verification passes.

## Acceptance Criteria

- Review completed and real findings fixed or documented.
- Backend and frontend automated checks pass.
- Verification file has `status: passed` or explicitly documents `human_needed/gaps_found/blocked`.
- Phase 05 requirements are marked complete only on pass.

## Plan-Checker Self-Review

- Closure does not start Phase 06.
- Human approval is not fabricated.
- Persistent state reflects actual verification status.
