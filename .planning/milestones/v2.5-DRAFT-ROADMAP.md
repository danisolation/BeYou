## Phases

### Phase 74: Design System Foundation

**Goal:** Mở rộng presentation-only primitives + typography/motion/dark-mode tokens cho toàn app reskin.
**Requirements:** UX-DS-01, UX-DS-02, UX-DS-03, UX-DS-04
**References:** shadcn/ui, Radix UI, Linear, Be Vietnam Pro
**Success criteria:**

1. `ui-primitives.tsx` có Sheet, Command, Tabs, Tooltip, Progress, Dialog — không import business logic.
2. Be Vietnam Pro + Inter loaded, typography scale tokenized.
3. Motion tokens trong `tailwind.config.ts`, `prefers-reduced-motion` respected.
4. Dark mode contrast WCAG AA pass cho all role shells.
5. Phase 34 primitive contract tests vẫn pass + mở rộng cho primitives mới.

### Phase 75: Student Home + Mood + Nav Reskin

**Goal:** Calm/Daylio/Material FAB pattern cho student daily entry surface.
**Depends on:** Phase 74
**Requirements:** UX-STU-01, UX-STU-02, UX-STU-03, UX-STU-08
**References:** Calm "Today", Daylio, Finch, iOS Emergency SOS, Material FAB
**Success criteria:**

1. Home dashboard render Calm-style stack (max 3 daily cards), no guilt copy.
2. Mood check-in full-screen sheet với 5 emoji + tags, haptic on tap.
3. Bottom nav 5-tab + center floating SOS, hold-1.5s-to-confirm.
4. SOS post-send reassurance screen với 3 recommended actions.
5. Existing SOS backend flow + adult notification path unchanged (Phase 4/35 tests pass).

### Phase 76: Student Chat + Scenario + Self-check Polish

**Goal:** Pi/Duolingo-Stories/Duolingo-lesson pattern cho 3 student learning/conversation surfaces.
**Depends on:** Phase 75
**Requirements:** UX-STU-04, UX-STU-05, UX-STU-06, UX-STU-07
**References:** Pi (Inflection), ChatGPT mobile, Duolingo Stories, Duolingo lessons, Apple Notes
**Success criteria:**

1. Chat: slide-over history drawer, suggestion chips, distress visual banner (backend safety unchanged).
2. Self-check: 1-question-per-screen + progress bar + soft result card.
3. Scenario: chat-bubble render + choice chips + Peerlight feedback bubble.
4. Support plan: inline-edit card sections + explicit share preview.
5. Snapshot-based historical attempts still interpretable (Phase 3 invariants).

### Phase 77: Teacher Portal Linear Reskin

**Goal:** Linear Issues + MyChart tabs pattern cho teacher support workflow.
**Depends on:** Phase 74
**Requirements:** UX-TCH-01, UX-TCH-02, UX-TCH-03, UX-TCH-04
**References:** Linear Issues, MyChart clinician view, Intercom inbox, Headspace empty states
**Success criteria:**

1. Student-with-SOS list dense Linear-style, SOS-only visibility unchanged (Phase 35 tests pass).
2. Support summary tabs (Tổng quan/Mood aggregate/Shared notes/Hành động), reason-gate unchanged (Phase 24).
3. SOS action workflow 3-status + keyboard shortcuts + audit trail visible.
4. Empty state reassurance copy.
5. No raw transcript, no risk leaderboard, no per-student drilldown beyond Phase 24/35 contract.

### Phase 78: Parent Portal Apple-Family Reskin

**Goal:** Connection-first tone, read-only sharing dashboard, không surveillance.
**Depends on:** Phase 74
**Requirements:** UX-PAR-01, UX-PAR-02, UX-PAR-03, UX-PAR-04 (UX-PAR-04 optional gated)
**References:** Apple Family, Apple Health Sharing, iOS critical alert
**Success criteria:**

1. "Con bạn đang chia sẻ với bạn" framing + warm copy review.
2. Read-only dashboard list consented items only (Phase 23 contract unchanged).
3. SOS notification full-screen modal với cách-nói guidance.
4. UX-PAR-04 chỉ implement nếu approved + không tạo new notification channel.
5. Privacy boundary tests (Phase 23/24) pass.

### Phase 79: Admin Linear Shell + Command Palette + Editor Preview

**Goal:** Linear app shell + cmdk palette + Sanity-style content editor preview.
**Depends on:** Phase 74
**Requirements:** UX-ADM-01, UX-ADM-02, UX-ADM-03, UX-ADM-04, UX-ADM-05
**References:** Linear, Vercel dashboard, Stripe dashboard, Sanity Studio, Plausible, cmdk
**Success criteria:**

1. Admin shell Linear-pattern với fixed sidebar + dark mode.
2. ⌘K command palette navigate/search (no raw data search).
3. Operations dashboard cards Vercel-style, metadata-only unchanged (Phase 36 contract).
4. Content editor wizard 3-step + live preview pane.
5. Reports aggregate chart kit, no per-user drilldown.

### Phase 80: Dark Mode Parity Full App

**Goal:** Dark mode coverage 100% all role shells + components, contrast audit.
**Depends on:** Phase 75, 77, 78, 79
**Requirements:** UX-DS-04, UX-A11Y-01 partial
**Success criteria:**

1. All routes render correctly in dark mode (visual regression baseline).
2. WCAG AA contrast pass via automated check.
3. Theme persistence across sessions (cookie, no localStorage of sensitive data).

### Phase 81: UI/UX Release Gates + A11y + Visual Regression

**Goal:** Final gate phase — axe-core CI, keyboard nav audit, Playwright visual diff, full release verification.
**Depends on:** Phase 80
**Requirements:** UX-A11Y-01, UX-A11Y-02, UX-A11Y-03, UX-A11Y-04, UX-A11Y-05
**Success criteria:**

1. axe-core CI gate green cho 4 role dashboards + critical flows.
2. Keyboard nav audit script pass (no trap, all interactive reachable).
3. Playwright visual regression baseline + diff threshold tuned.
4. Backend pytest, frontend vitest, lint, build, smoke:demo, smoke:pilot, release-gates ALL green.
5. v1.5-v2.4 privacy/security tests vẫn pass (no regression).

## Risks

| Risk                                                       | Mitigation                                                                 |
| ---------------------------------------------------------- | -------------------------------------------------------------------------- |
| Reskin có thể break Phase 35 visual walkthrough            | Re-run + update walkthrough at end của mỗi role-reskin phase               |
| Be Vietnam Pro font loading impact LCP                     | Subset + `font-display: swap` + preload                                    |
| Command palette index có thể expose route names lộ feature | Manual allowlist, no auto-discovery                                        |
| Dark mode contrast regressions                             | Automated axe-core trong Phase 81 gate                                     |
| Visual regression flakiness                                | Playwright `maxDiffPixelRatio` tuning + masked regions for dynamic content |
| Scope creep (8 phases lớn)                                 | Strict requirements traceability, no new features ngoài 30 reqs            |

## Non-goals

- Không thay đổi backend API/auth/privacy.
- Không thêm tracking/analytics raw data.
- Không multi-school tenant UI (defer v2.6+).
- Không OAuth callback UI (defer per Phase 30 contract).
- Không mobile native app (PWA đủ cho v2.5).
- Không gamification/streak guilt mechanics.
