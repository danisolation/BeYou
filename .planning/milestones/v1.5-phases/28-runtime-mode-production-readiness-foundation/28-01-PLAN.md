<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| public caller -> /health/ready | Unauthenticated public readiness request must not receive runtime mode, checks, config, or secrets. |
| admin user -> /api/admin/operations/readiness/dashboard | Authenticated admin receives metadata-only detail and must not receive sensitive values. |
| environment variables -> readiness/admin responses | Sensitive deployment config crosses into application logic and must be reduced to safe status metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-28-01 | Information Disclosure | `public_readiness_from_report` | mitigate | Keep `PublicReadinessResponse` as only `status` and `generated_at`; never add runtime mode or checks to public readiness. |
| T-28-02 | Information Disclosure | `build_operations_dashboard` | mitigate | Replace raw origin and cookie-name output with booleans/counts/kinds; add tests rejecting secrets, cookies, connection strings, credentials, token-like values, raw emails/IDs, and raw student data. |
| T-28-03 | Spoofing | `RUNTIME_MODE` settings | mitigate | Use `Literal["local_demo", "public_demo", "production_pilot"]`; invalid values fail validation instead of silently mapping to production. |
| T-28-04 | Tampering | Production pilot config | mitigate | `runtime_environment_compatibility`, `demo_seed_policy`, `demo_login_policy`, `origin_security`, and `cookie_security` fail unsafe combinations. |
| T-28-05 | Elevation of Privilege | Admin readiness details | mitigate | Keep admin readiness behind existing `_require_admin_operations` role and permission gate; this plan does not add public detail routes. |
| T-28-06 | Information Disclosure | Provider/SMTP/database config | mitigate | Readiness summaries/remediations describe status and remediation only; never echo env values or secret names with values. |
</threat_model>
