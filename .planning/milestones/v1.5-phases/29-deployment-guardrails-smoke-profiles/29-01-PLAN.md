<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| operator env -> guardrail CLI | Local/deploy environment variables enter Node script. |
| repo config -> guardrail parser | `render.yaml` and `frontend/vercel.json` are parsed as deployment evidence. |
| CLI output -> logs/terminal | Guardrail output may be copied into issue logs or operations notes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-29-01-01 | I | deployment-guardrails.mjs output | mitigate | Print env var keys and remediation only; tests reject secret/cookie/raw labels and values. |
| T-29-01-02 | T | Render/Vercel config validation | mitigate | Parse YAML/JSON and assert exact `backend`, `/health/live`, `nextjs`, `npm run build`, `npm install`. |
| T-29-01-03 | S | API target mismatch | mitigate | Production pilot fails if `NEXT_PUBLIC_API_BASE_URL` does not exactly match `BEYOU_EXPECTED_BACKEND_URL`. |
| T-29-01-04 | R | false pass evidence | mitigate | Exit 1 on any `fail`; include deterministic pass/warn/fail records from pure validators. |
</threat_model>
