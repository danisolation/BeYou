# Phase 22 Plan: Student reminder preferences, in-app reminder card, and no-auto-SOS regression coverage

**Created:** 2026-05-22  
**Status:** Complete

## Goal

Let students manage in-app mood reminder consent, quiet hours, pause/resume, and optional reminders while preserving BeYou's non-clinical and no-auto-SOS boundaries.

## Tasks

- [x] Add student notification preference API:
  - `GET /api/student/notification-preferences`
  - `PUT /api/student/notification-preferences`
- [x] Add student mood reminder API:
  - `GET /api/student/reminders/mood-check-in`
  - `POST /api/student/reminders/mood-check-in/dismiss`
  - `POST /api/student/reminders/mood-check-in/snooze`
  - `POST /api/student/reminders/mood-check-in/open`
- [x] Add reminder eligibility logic for explicit consent, in-app channel, quiet hours, pause, snooze, cadence, and recent check-ins.
- [x] Add student notification preference page with channel boundary, quiet-hour, cadence, pause, and resume controls.
- [x] Add optional dashboard reminder card with dismiss/snooze/open actions and supportive copy.
- [x] Add backend and frontend regression tests for preference controls, reminder actions, no SOS/adult notification, and external channel rejection.

## Risk Controls

- Reminder routes use student role + privacy acknowledgement gates.
- Mutations require same-site mutation protection.
- External channels remain rejected server-side.
- Reminder service does not call SOS or adult notification services.
- Reminder audit metadata records status/count decisions only.
