# Quick Task 260601-ek4: Redesign entire app UI to match new Peerlight AI design

**Date:** 2026-06-01
**Type:** Visual rebrand (token-level, cascades across all roles)

## Goal

Redesign the whole app's look-and-feel to match the provided Peerlight AI dashboard
mockup: violet/cyan pastel palette, soft gradient background, rounded soft-shadow
cards, gradient sidebar active states, and a gradient brand mark.

## Key insight

The frontend already drives nearly all color from design tokens
(`primary`, `surface`, `outline-variant`, `on-background`, …). Only `globals.css`
and `tailwind.config.ts` hold the hardcoded teal "Stitch" palette. Re-skinning the
tokens cascades the new theme to **every** page and role (student/teacher/parent/admin)
with minimal, low-risk edits.

## New palette (from mockup)

- ink `#17204c`, muted `#6d7394`, line `#e8ebfb`
- violet `#7457e8` (primary), violet-2 `#9178ff`, blue `#65aefc`, cyan `#85e9f1`, pink `#f36bac`
- soft bg gradient `linear-gradient(135deg,#f7fbff,#fbf8ff,#f5f9ff)`
- soft shadow `0 10px 28px rgba(68,76,150,.10)`

## Changes

1. `tailwind.config.ts` — re-skin `primary*`, `surface*`, `secondary`, `tertiary`,
   accent aliases to violet/cyan; add `accent-blue/cyan/pink/violet`.
2. `app/globals.css` — `:root` + `.dark` variables to violet family; new body
   gradient; selection color; add gradient utility classes (`.brand-gradient`,
   `.nav-active`, `.cta-gradient`, `.soft-card`).
3. `components/navigation/{student,teacher,parent,admin}-sidebar.tsx` — gradient
   active nav state (`.nav-active`).
4. `app/(authenticated)/layout.tsx` — gradient brand mark in the top header.
5. `components/stitch-card.tsx` + `components/ui-primitives.tsx` — richer soft
   shadow + 20px radius; gradient primary CTA.
6. `app/(authenticated)/student/page.tsx` — gradient hero welcome banner (flagship
   reference for the mockup look).

## Verification

- `npm run lint` clean
- `npm run build` succeeds
- Spot-check token cascade (no leftover teal references outside intentional spots)
